# Contributing Guidelines: The AI-Driven Evolution

Willkommen in diesem Repository. Dies ist kein gewöhnlicher Ort für handgetippten Code, sondern ein Labor für die nächste Stufe der Software-Genese. Wir verstehen uns als Kuratoren von **Interpolations-Maschinen** und sammeln Code, dessen Entstehungsprozess die Grenzen zwischen Geist und Algorithmus auslotet.

Um die Integrität des Projekts und die Resonanz zum **TSM nach Rosi Hach** zu wahren, gelten folgende Richtlinien:

---

## 1. AI-First Paradigma (Hauptzweck)
Dieses Repository dient primär der Sammlung von Programmen, die durch Künstliche Intelligenz (LLMs, generative Modelle) erstellt wurden.
* **Erwünscht:** Code, der durch präzises Prompting und iterative Verfeinerung entstanden ist.
* **Hintergrund:** Wir untersuchen hier die „Deepfakes der Logik“ – wie KI Strukturen erschafft, die über das rein menschliche Maß an quantitativer Reproduktion hinausgehen.

## 2. Themenrelevanz & Kontext
Erkenntnis bleibt **t-gebunden** und kontextlos, wenn sie nicht auf ein Ziel ausgerichtet ist. Jeder Beitrag muss einen direkten Bezug zum Thema des Repositorys aufweisen.
* Adressiere Kernfragen des Projekts (z. B. TSM-Anwendungen, Strukturanalysen oder prozedurale Resonanz).
* Beiträge ohne klaren Themenbezug werden als „rauschende Korrelation“ abgelehnt.

## 3. Selbst-dokumentierender Code (Struktur-Klarheit)
Gemäß der **Structured Resonance** muss der Quellcode in sich schlüssig sein. Er ist die „Säule“, die ohne äußere Stützen stehen muss.
* **Variable & Funktionen:** Nutze sprechende Namen, die die Logik unmittelbar offenbaren.
* **Architektur:** Die Form muss der Funktion folgen. Ein gut geschriebener KI-Code benötigt keine Kommentare, die das Offensichtliche erklären. Er beweist sich durch seine Ausführung.

## 4. Erlaubte Sprachen & Umgebungen
Um die technische Resonanzfläche konsistent zu halten, akzeptieren wir ausschließlich:
* **C / C++:** Für die fundamentale, maschinennahe Logik.
* **Python:** Inklusive Skripte für **Blender** (prozedurale Strukturen).
* **Processing (Java/JS):** Für die Darstellung von Schwingungen und visuellen Resonanzen.

## 5. Der "Minimal-Interaktions-Standard" (Begleitdokumentation)
Zusätzlich zum Code ist eine `README.md` im Unterordner deines Beitrags zwingend erforderlich. Sie umfasst drei Säulen:
1. **Korrespondierendes Thema:** Welcher Aspekt des TSM/GAL wird adressiert? (z.B. "Simulation von $V_r(t)$").
2. **Bedienung (Kurzform):** Wie wird die Maschine gestartet? (z.B. `python script.py`).
3. **Strukturelle Innovation:** Ein präziser Hinweis auf die **„KI-Signatur“**. Wo hat die KI eine Lösung gefunden, die über menschliche Standard-Logik hinausgeht? Markiere die Stelle im Code kurz mit `# AI-Innovation` oder `// AI-Innovation`.

## 6. Inspirations-Matrix (5x3-Regel)
Um die "zwanghafte Extrovertiertheit" in produktive Bahnen zu lenken, muss jeder Beitrag durch externe Quellen fundiert sein. Wir erwarten:
* Genau **5 Themenbereiche** mit jeweils **3 hochwertigen Links** (Webinhalte), die das behandelte Thema vertiefen oder als Inspiration dienten.

## 7. Einreichungsprozess
1. **Forke** das Repository.
2. **Branch:** Erstelle einen Branch mit dem Namen des KI-Modells (z.B. `feature/gpt4-resonance`).
3. **PR:** Dokumentiere im Pull Request kurz deinen methodischen Ansatz (Prompt-Strategie).
4. **Lizenz:** Mit der Einreichung bestätigst du die Veröffentlichung unter der **MIT-Lizenz**.

---

## Checkliste für Contributoren
- [ ] Code in C/C++, Python oder Processing?
- [ ] Hauptanteil durch KI generiert?
- [ ] Code ist selbstdokumentierend (klar benannte Variablen/Strukturen)?
- [ ] `README.md` im Unterordner (Thema, Bedienung, KI-Innovation)?
- [ ] Inspirations-Matrix (5 Themen à 3 Links) vollständig?
- [ ] MIT-Lizenz akzeptiert?
